## Expand SPPM feature showcase coverage

Update the showcase example so it demonstrates the full set of supported SPPM features, including newly added semantic capabilities.

## Why

The feature showcase is the most visible reference artifact for SPPM output. As support expands, the example should demonstrate the supported notation set clearly enough to act as a living regression and communication artifact.

## Acceptance Criteria

- The showcase example exercises the major supported SPPM semantic features.
- Rendered reference artifacts are updated alongside the example.
- The example remains readable and does not become a random pile of disconnected features.
- Tests or reference checks cover the intended output shape where practical.

## Notes

Prefer one coherent example over multiple disconnected micro-features unless the example becomes visually overloaded.
