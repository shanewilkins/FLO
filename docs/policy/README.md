# FLO Policy

This directory holds normative project policy for FLO.

Policy documents answer questions like:

- Which artifacts are authoritative.
- Which documents are normative versus explanatory.
- What must change when language semantics or diagram behavior changes.

Current policy documents:

- `authoritative_artifacts.md`
- `artifact_scope.md`
- `sppm_regression_gates.md`

Related design companion:

- `docs/design/artifact_taxonomy.md` explains FLO-owned artifact families,
  canonical versus derived outputs, and artifact lifecycle guidance after a
  proposal has already been judged in-scope by policy.

Policy documents are intentionally short. They should define rules and
decision boundaries, not restate implementation details.
