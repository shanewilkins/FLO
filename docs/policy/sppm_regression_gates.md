# FLO SPPM Regression Gates Policy

Purpose: define the required CI gates, frozen strategy policy, and intentional update workflow for canonical SPPM render outputs.

## Canonical artifacts

The canonical SPPM corpus manifest is `examples/conformance/sppm_corpus.json`.

The checked-in accepted SPPM baseline lives under `renders/conformance/sppm_baseline/`.

The historical strategy matrix scoreboard is generated by `scripts/run_sppm_strategy_matrix.py`.

The locked production strategy lives in `src/flo/render/layout_core/sppm_strategy.py`.

## Required CI gates

Every change that can affect SPPM geometry or diagnostics must pass the named SPPM invariant suite and the baseline drift gate.

Routine CI also runs a fast determinism-focused test slice for known ordering and drift primitives.

The SPPM invariant suite must prove that the canonical corpus still satisfies the layout invariant assertions.

The baseline drift gate must prove that regenerating the canonical SPPM baseline produces byte-identical artifacts.

Generated SPPM artifacts must never be edited by hand.

## Frozen strategy policy

The promoted SPPM strategy is now locked.

Normal correctness and determinism work must validate against the accepted baseline rather than rerunning the full strategy matrix.

The strategy matrix is retained only as a historical comparison tool or a deliberate redesign tool.

Do not add the strategy matrix back to routine CI or pre-commit flows unless the project explicitly reopens strategy exploration.

## Required commands

Run the baseline drift gate with:

```bash
uv run python scripts/check_sppm_baseline_drift.py
```

Run the SPPM invariant suite with:

```bash
uv run pytest -q tests/unit/test_sppm_layout_invariants.py
```

Run the focused determinism regression slice with:

```bash
uv run pytest -q \
  tests/unit/test_sppm_baseline_drift.py \
  tests/unit/test_layout_core_elk_requests.py \
  -k 'parent_only_subprocess_edges_are_stable or test_sppm_baseline_drift'
```

## Intentional baseline update workflow

Use this workflow only when a reviewed change intentionally alters accepted SPPM geometry, diagnostics metadata, or the locked production strategy.

1. Regenerate accepted baseline artifacts with `uv run python scripts/build_sppm_baseline_artifacts.py --clean`.
2. Review the resulting diffs in `renders/conformance/sppm_baseline/` and confirm every changed file is expected.
3. Re-run `uv run python scripts/check_sppm_baseline_drift.py` and require a clean pass before merge.
4. Update user-facing or policy documentation in the same change when the accepted artifact meaning or workflow changed.

If the change intentionally reopens strategy selection, run the matrix manually, review the resulting scoreboard, and document why the strategy freeze is being changed.

## Change-review rule

Any pull request that updates `renders/conformance/sppm_baseline/` must explain why the drift is intentional and what review evidence was used to accept it.
